# 테스트 가이드

게임 실행 방법은 [README.md](README.md), 구현 단계는 [DEVELOPMENT.md](DEVELOPMENT.md)를 참고하세요.

## 자동 테스트 (단위 테스트)

순수 게임 로직(`src/js/game/`)을 Node.js 내장 테스트로 검증합니다.

### 사전 요구

- [Node.js](https://nodejs.org/) 18 이상 (`node -v`로 확인)

### 실행

프로젝트 루트에서:

```bash
npm test
```

또는:

```bash
node --test tests/**/*.test.js
```

모든 테스트가 통과하면 종료 코드 0으로 끝납니다.

## 수동 테스트 (브라우저)

화면·버튼·DOM/CSS Grid 렌더링(inline style)은 브라우저에서 직접 확인합니다.

### 실행

1. `src/index.html`을 브라우저에서 엽니다.
2. ES module을 사용하므로, `file://`에서 동작하지 않으면 Live Server 등으로 `src` 폴더를 서빙합니다.

   ```bash
   npx serve src
   ```

3. 표시된 주소(예: `http://localhost:3000`)로 접속합니다.

### 단계 0 체크리스트

| # | 확인 항목 | 기대 결과 |
|---|-----------|-----------|
| 1 | 게임 보드 | 10×20 CSS Grid, 빈 칸(어두운 배경)만 표시 |
| 2 | 점수 | `0` 표시 |
| 3 | 시작 버튼 | **시작** 텍스트 표시 |
| 4 | 시작 클릭 | 버튼이 **재시작**으로 변경 |
| 5 | 재시작 후 | 보드·점수 초기화 |
| 6 | 조작법 | 키 안내 문구 표시 |

### 단계 1~2 체크리스트

| # | 확인 항목 | 기대 결과 |
|---|-----------|-----------|
| 1 | npm test | board + tetromino 테스트 전부 통과 |
| 2 | 게임 보드 | canvas 없음, 10×20 div grid |
| 3 | 빈 보드 | 모든 칸 동일한 빈 칸 색 |
| 4 | 시작 클릭 | 상단 중앙에 테트로미노 1개 (T) |
| 5 | 재시작 | 보드·점수 초기화 후 다시 블록 표시 |
| 6 | 키·낙하 | 입력·자동 하강 없음 |

## 테스트 대상 구분

| 구분 | 위치 | 방법 |
|------|------|------|
